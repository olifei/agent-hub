# Agent Hub

Production-ready AI agent templates for Google Cloud Agent Engine.

## Deploy an Agent

```bash
git clone https://github.com/olifei/agent-hub
cd agent-hub/<agent-name>
bash deploy.sh <YOUR_PROJECT_ID>
```

## Contributing an Agent

### Prerequisites

- A working ADK agent with `root_agent` defined in `agent.py`
- A GCP project for deployment testing
- [Claude Code](https://claude.ai/code) installed
- [uv](https://docs.astral.sh/uv/) installed

### Step 1: Run the `/portal-ready` skill

In your agent project directory, run Claude Code with this repo as an additional directory:

```bash
cd /path/to/your-agent
claude --add-dir /path/to/agent-hub
```

Then type `/portal-ready`. The skill will guide you through:

1. **Generating `agent.yaml`** — Portal display metadata (name, description, industry, tags)
2. **Scanning GCP dependencies** — Detect BigQuery, GCS, Firestore, etc.
3. **Generating `setup.sh`** — GCP resource creation script (if needed)
4. **Configuring `pyproject.toml`** — Adding `[tool.agent-starter-pack]` config
5. **Running `agents-cli scaffold enhance`** — Adding deployment infrastructure
6. **Generating `deploy.sh`** — One-click deployment script

Review and confirm at each step.

### Step 2: Test deployment

Deploy to your own GCP project to verify everything works:

```bash
bash deploy.sh <YOUR_PROJECT_ID>
```

Confirm the agent is functional in the Agent Engine Playground.

### Step 3: Submit a PR

Once deployment is verified, submit a PR to this repo with your agent directory.

### Required files

| File | Required | Description |
|------|----------|-------------|
| `agent.yaml` | Yes | Portal metadata (name, displayName, description, industry, tags) |
| `deploy.sh` | Yes | One-click deployment script |
| `<agent_dir>/agent.py` | Yes | Agent code with `root_agent` defined |
| `<agent_dir>/agent_runtime_app.py` | Yes | Agent Engine entrypoint (generated by agents-cli) |
| `pyproject.toml` | Yes | With `[tool.agents-cli]` or `[tool.agent-starter-pack]` config |
| `setup.sh` | If needed | GCP resource setup (BigQuery, GCS, etc.) |
| `README.md` | Recommended | Agent documentation |

### setup.sh requirements (if applicable)

If your agent depends on GCP services, `setup.sh` must:

- Accept `PROJECT_ID` as first argument, `REGION` as optional second
- Support `--cleanup` flag for resource removal
- Have `set -e` for error handling
- Enable required GCP APIs
- Set up IAM roles
- Be idempotent (safe to run multiple times)
- Print a cost warning before creating paid resources
- Output resource info for `.env` at the end
- Only use tools available in Cloud Shell

### What NOT to submit

- Agents that require ML model training or large model downloads
- Agents with hardcoded project IDs, API keys, or credentials
- Agents that have not been successfully deployed and tested
